# 홖 — 손으로 만지는 것들

두 작품을 손으로 조작한다.

| 키 | |
|---|---|
| **1** | 입자 분사 |
| **2** | 실뭉치 (우상단 ⚙ 로 조절 패널) |
| **Tab** | 전시 모드 — UI·카메라 미리보기·커서를 전부 숨긴다 (손 인식은 계속 돈다) |

- 손 이동 = 마우스 이동
- **주먹 쥐기** — 입자에서는 분사(*흩다*), 실뭉치에서는 **한 움큼 잡기**(*치대다*):
  잡은 채 끌면 실이 늘어나고, 펴는 순간 샤라락 튕겨 돌아간다.
  쥔 세기가 아날로그다 — 꽉 쥘수록 뭉치가 조여든다. 양손으로 잡고 벌리면 실뜨기.
- 카메라에서 **멀어지면 전체가 작아진다** (멀어서 작게 보이는 것처럼)
- 마우스/터치는 그대로 살아 있다 — 손 없이도 작동한다

<https://greenteavillain.github.io/void/>

## 실행

```bash
bash start.sh          # → http://localhost:8745
```

**`file://` 로 열면 카메라가 절대 안 된다.** `getUserMedia` 는 보안 컨텍스트를 요구하는데
Chrome 은 `file://` 을 opaque origin 으로 취급한다. 반드시 `localhost` 또는 https 로.
**Safari 말고 Chrome.** Safari 는 MediaPipe WebGL 이슈가 열려 있고 노출 제어도 없다.

MediaPipe(18MB)는 **✋ 버튼을 누를 때만** 받는다 — 그냥 구경하는 방문자는 안 받는다.

**껍데기가 카메라를 쥔다.** 장면마다 페이지를 나누면 전환할 때마다 MediaPipe를 다시 초기화하고
카메라를 다시 열어야 한다 — 1초쯤 끊기고 보정값(여기가 기준)도 날아간다.

**★ `sample()` 은 프레임당 딱 한 번만 부른다.** 이 함수는 부를 때마다 렌더용 위치를 시간에 따라
굴리므로(톱니 제거), 작품·HUD·미리보기가 각자 부르면 같은 일을 세 번 하게 된다.
껍데기가 한 번 뽑아서 나눠준다 — 실뭉치는 어트랙터(손바닥+손끝), 입자는 손 목록(`palmsOf`).

**전시장으로 옮길 때는 `hand/hand-source.js` 만 갈아끼우면 된다.** `sample()` 이 같은 모양을
뱉기만 하면 두 작품 다 손도 안 댄다. (암전이면 IR 블롭 트래킹으로 → 아래 참고)

## 만들면서 알아낸 것들

각 항목은 **측정으로** 확인했다. 재측정하지 말 것.

### 손 인식

- **워커 필수** — `detectForVideo()` 는 동기 반환(Promise 아님). 17.5ms > 60fps 예산 16.7ms.
  메인스레드에서 부르면 조용히 30fps로 반토막.
- **모듈 워커에선 `forVisionTasks(path, true)`** — 두 번째 인자 없으면 `ModuleFactory not set.` 로 죽는다.
  기본값은 classic script 글루를 `importScripts` 로 넣으려 하는데 모듈 워커에선 그게 TypeError고,
  폴백한 `import()` 로는 `var ModuleFactory` 가 전역에 안 잡힌다. `true` 면 모듈 빌드 글루를 쓰고
  그건 끝에서 `globalThis.ModuleFactory` 를 명시 세팅한다.
- **`runningMode: 'VIDEO'`** 대문자. 공식 샘플의 `'video'` 는 타입 정의와 다르다.
- **손 정체성은 handedness 로 하면 안 된다** — 공식 테스트 이미지(left_hands.jpg)만 해도 두 손 다 "Left".
  위치 기반 최근접 매칭을 쓰되, **마지막 위치가 아니라 속도로 예측한 위치**에 대고 잰다.

### 움켜쥐기

`openness = 평균(|손끝−손목| / |중지MCP−손목|)`, **worldLandmarks(3D)** 기준.
2D로 재면 손을 카메라로 향했을 때 편 손도 주먹으로 보인다(원근 단축).
분모는 주먹 쥐어도 안 변하는 손바닥 뼈라 기준자로 적합하다. 엄지는 뺀다(가장 노이즈가 크다).

공식 제스처 사진 실측: **주먹 0.812** · ☝️ 1.024 · ✌️ 1.407 · **편 손 1.747**.
palm 크기가 0.058~0.105m(1.8배)로 달라도 openness는 일관 = **비율이라 크기 불변**.
임계값 1.15/1.40 (양쪽에서 등거리, 슈미트 트리거로 점멸 방지).

### 거리 = 크기

찍힌 손바닥 크기(`|손목−중지MCP|`, **매핑 전 정규화 이미지 좌표**, 화면비 보정)가 곧 거리.
z는 필요 없다 — 원근이란 게 "멀면 작게 보인다"이므로 보이는 크기가 이미 거리다.
**전부 같은 수로** 곱한다: 링 반경 · 속도 · 입자 크기. 개수와 수명은 안 건드린다.
기준 거리는 카메라 화각마다 달라 코드가 못 정한다 → **여기가 기준** 버튼.

### 속도는 주사율과 무관하다

두 작품 다 원래 **프레임 기준**이었다 — `p.x += p.vx` 에 dt가 없어서 화면 주사율이 곧 작품 속도.
실측: 60Hz 893px/초 · 120Hz 1787 · 144Hz 2144. **갤러리 프로젝터는 대개 60Hz다.**
60Hz 거동을 기준으로 못박았다(`k = dt/16.67`). k=1이면 원본과 완전히 동일.
★ 실뭉치의 **감쇠는 지수** — `Math.pow(D, k)`. `D*k` 로 곱하면 고주사율에서 감쇠가 사라져 발산한다.

### "빨리 움직이면 순간이동" — 범인은 1€ 필터의 beta

`beta` 는 **신호의 단위에 종속**인데 논문 예제값 `0.007` 을 그대로 썼다. 그건 픽셀 신호용이다.
여기 신호는 정규화 좌표(0~1)라 빠른 휘두름이 초당 2.0 → 컷오프 = `1.0 + 0.007×2 = 1.014Hz`
= **속도 적응이 사실상 0** = 그냥 1Hz 고정 저역통과. 지연만 받고 이득은 0.

전 구간 실측(검출 30/초 → 렌더 120fps, 외삽 포함): **0.007 → 540px 뒤처짐** · 3 → 151 ·
**10 → 109 (채택)** · 20 → 90 · 40 → 72.
★ **정지 시 떨림은 beta와 무관**(1.10 → 1.29px) — 속도가 0이면 beta가 안 곱해진다. **올리는 데 대가가 없다.**

### 톱니 — "마우스로 끌면 자연스러운데 주먹으로 끌면 끊긴다"

**주사율 탓이 아니다.** 마우스는 매 프레임 진짜 위치, 손은 초당 30번만 진짜 + 나머지는 예측.
검출이 올 때마다 위치를 필터값으로 리셋하면, 필터는 떨림을 깎느라 뒤처져 있으므로 그 리셋이
곧 뒤로 당김 → 초당 30번 톱니. 경로 분사가 "그 프레임의 이동거리"에 입자를 뿌리므로
**이동거리 출렁임 = 밀도 출렁임 = 띠**.

고침: 렌더용 위치를 **속도로 굴리고 오차만 시상수(70ms)로 흡수**.
실측(등속 손, 프레임당 12px가 정답): 들쭉날쭉 **1.83 → 0.79px** (7.0~16.9 → 10.6~13.7),
**지연은 오히려 감소**(20.4 → 19.3px) — 속도가 움직임을 만드니 공짜.

### 움켜쥐기 = 치대기 (실뭉치)

편 손은 헤치고(*젓다*), 쥔 손은 잡는다(*치대다*) — 손 모양이 동사를 바꾼다.

주먹을 쥐면 반경(150 월드) 안의 점들이 잡힌다: 스프링 목표가 제자리 대신
**주먹 + 잡던 순간의 오프셋 × 오므림**이 된다.
- **잡는 순간 아무것도 안 튄다** — 오프셋이 현재 위치 기준이라 잡는 순간엔 목표=현위치.
- **오므림이 아날로그** — openness [0.81, 1.15] → 배율 [0.35, 1.0]. 꽉 쥘수록 뭉치가 조여든다
  (실측: 퍼짐 66 → 19).
- **놓는 순간이 보상** — 플래그만 지우면 기존 스프링이 알아서 샤라락 복귀시킨다.
- 잡힌 점은 부유·밀어내기를 받지 않는다 — 쥐고 있는 손이 동시에 밀쳐내면 힘이 싸운다.
- 검증: 연속 끌기 60프레임 동안 뭉치-주먹 거리 중앙 6.1/최대 16(월드), 놓친 프레임 0,
  양손 독립(4,124 + 672점). 순간이동급 점프(한 프레임에 0.23)로 트랙이 갈리면
  놓고 새 자리에서 다시 잡는다 — 유령 없이 우아하게 실패.

### 경로 분사

한 프레임에 링 하나를 "지금 위치"에 통째로 뱉으면, 분사점이 움직일 때 프레임마다 링이 딴 자리에
찍혀 **뚝뚝 끊긴 웨이브**가 된다. 입자를 프레임 동안 지나간 **선분에 시간순으로** 뿌린다.
선분의 30% 지점 = 프레임 시간의 30% 시점이므로 **근사가 아니라 정확한 계산**이다.
(대안인 "시뮬레이션 4배 잘게 돌리기"는 비용 4배에 여전히 근사)
실측: 분사점 속도 4배 → 퍼짐도 정확히 4배(5.4 → 21.6px).

### 죽은 가설 (재측정 금지)

- **모션 블러 아님** — 추적은 손 너비 55%(110px)가 번져도 두 손 다 유지. 주먹 openness도 0.778→0.827 안정.
  (단 **놓친 뒤 재획득은 45px 블러부터 무너진다** — 60px→한 손만, 110px→0)
- **신뢰도 임계값 무효** — 0.5 / 0.3 / 0.2 결과 완전 동일.
- **해상도 무효** — MediaPipe가 내부에서 어차피 고정 크기로 줄인다.
- **워커 2개 무의미** — 카메라가 30fps 천장이라 버림이 7%뿐. 두 번째 워커가 벌어올 게 7%.

## 남은 것

1. **모니터 144Hz** — 주 화면(LG ULTRAGEAR 2560×1440)이 60Hz로 돌고 있다. 그 해상도에서 60/120/144
   전부 지원한다(측정 확인). `시스템 설정 → 디스플레이 → 새로고침 빈도`.
   속도가 주사율과 무관해졌으므로 **올려도 부드러워지기만 하고 작품 속도는 그대로**다.
2. **실제 손으로 튜닝** — **여기가 기준**(거리) · **따라오기**(beta) 슬라이더. 기본값은 추정치다.
3. **전시장 조명이 결정적 변수** — 실뭉치는 검은 배경이라 암전이면 RGB 웹캠이 손을 못 본다
   (MediaPipe는 가시광만 학습, NIR 0장). 940nm IR 조명(850nm는 붉게 빛나 작품 오염) + IR 통과 필터,
   현장에서 반나절 테스트로 판가름. 실패 시 블롭 트래킹 폴백.
   NIR이 정보 부족한 게 아니다 — ThermoHands(SenSys'25): NIR 41.11mm < RGB 43.80mm.
4. **장시간 전시** — WebGL 컨텍스트 유실 대비 워치독은 넣었다(5초 무응답 시 워커 재생성).
   Chrome kiosk 모드 + Memory Saver에서 탭 제외할 것.

## 접은 것

- **터치디자이너** — 맥에서 자체 핸드트래킹이 **0**이다(Body/Face Track CHOP은 Windows+RTX 전용).
  맥 TD의 사실상 표준인 mediapipe-touchdesigner 플러그인이 하는 일 = 헤드리스 크롬에서 이 MediaPipe JS를
  돌려 웹소켓으로 TD에 쏘기(+3프레임 지연). Web Render TOP의 고속 경로는 Windows 전용이라 맥에선
  매 프레임 CPU 복사가 두 번. 비상업 라이선스는 1280×1280 캡(프로젝션 불가).
  → 멀티프로젝터+DMX 규모가 되면 그때 TD를 *쇼 컨트롤러*로만 쓰고 작품은 Syphon으로 넘길 것.
